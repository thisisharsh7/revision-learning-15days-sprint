## Interview Preparation

### Technical Questions to Prepare

**Python:**
- How do you handle errors when making HTTP requests?
- What's the difference between a list and a dictionary?
- How do you read and write JSON files?
- How do you structure a Python script for maintainability?

**React:**
- What are React hooks? Name the ones you know.
- When would you use useState vs useEffect?
- What is the virtual DOM?
- How do you prevent unnecessary re-renders?

**TypeScript:**
- What's the difference between `interface` and `type`?
- How do you type a function parameter?
- What is a union type?
- Why use TypeScript instead of JavaScript?

**Performance:**
- Why does rendering 10,000 items crash the browser?
- How does virtual scrolling solve this?
- What is memoization?
- When should you optimize performance?

**Routing Concepts:**
- What is regression testing?
- How do you determine if two routes are "different"?
- What is route geometry?
- What are turn-by-turn instructions?

### Project Questions to Prepare

**Architecture:**
- Walk through the data flow from GitHub Actions to web app
- Explain how route comparison works
- Why store results in S3 vs database?
- How would you handle 10,000 routes in the browser?

**Implementation:**
- How would you structure the React components?
- What state management would you use and why?
- How would you test the route comparison logic?
- What edge cases need handling?

**Timeline:**
- Is 12 weeks enough for this project?
- Which parts are highest risk?
- What would you build first?
- What's your testing strategy?

## Common Mistakes to Avoid

**Python:**
- Not handling network errors
- Forgetting to close files
- Hardcoding values instead of using variables
- Not logging errors for debugging

**React:**
- Mutating state directly
- Missing dependency arrays in useEffect
- Not typing props in TypeScript
- Creating new objects/functions on every render

**MapLibre:**
- Forgetting to load map before adding layers
- Not cleaning up map instance on unmount
- Using wrong GeoJSON format
- Not setting proper bounds

**General:**
- Trying to learn everything at once
- Not testing code as you write it
- Skipping documentation
- Not asking for help when stuck
